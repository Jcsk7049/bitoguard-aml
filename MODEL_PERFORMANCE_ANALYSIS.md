# BitoGuard 模型檢測成功率分析

## 當前性能指標

### 交叉驗證結果（5-Fold CV）
| 指標 | 平均值 | 標準差 | 範圍 |
|------|--------|--------|------|
| **Precision** | 9.83% | 8.12% | 0% ~ 18.18% |
| **Recall** | 0.43% | 0.36% | 0% ~ 0.91% |
| **F1 Score** | 0.82% | 0.70% | 0% ~ 1.73% |

### 問題分析

#### 1. **極低的 Recall（召回率）**
- 平均只有 **0.43%** 的黑名單用戶被正確識別
- 這意味著 **99.57% 的黑名單用戶被漏掉**
- 在 328 個黑名單用戶中，平均只抓到 1-2 人

#### 2. **低 Precision（精準度）**
- 平均 **9.83%** 的預測正例中才是真正的黑名單
- 大量誤報（False Positive）

#### 3. **根本原因**
- **數據不平衡**：黑名單 1,640 人 vs 正常用戶 49,377 人（比例 1:30）
- **特徵信息不足**：只有 3 個特徵
  - `twd_deposit_count`（台幣入金次數）
  - `twd_withdraw_count`（台幣出金次數）
  - `night_tx_ratio`（深夜交易比例）
- **模型欠擬合**：簡單特徵無法區分複雜的洗錢行為

## 改進建議

### 短期（快速改進）
1. **調整決策閾值**
   - 降低分類閾值（目前 0.5）→ 0.3 或更低
   - 增加預測正例的數量

2. **調整超參數**
   ```bash
   python main_pipeline.py --start-from train \
     --max-depth 8 \
     --eta 0.05 \
     --num-round 1000 \
     --scale-pos-weight 30
   ```

3. **使用 Glue 圖分析特徵**
   - 啟用 `glue_graph_hops.py` 計算多跳關聯
   - 加入圖特徵（超級節點、集群等）

### 中期（特徵工程）
1. **增加交易特徵**
   - 交易頻率、金額分布、時間模式
   - 帳戶年齡、首次交易時間

2. **增加圖特徵**
   - User-to-IP 多跳距離
   - 共享 IP 的用戶數
   - 交易網絡密度

3. **增加行為特徵**
   - 交易金額異常度
   - 交易對手集中度
   - 跨幣種交易比例

### 長期（模型改進）
1. **集成模型**
   - XGBoost + LightGBM + CatBoost
   - 加權投票

2. **異常檢測**
   - Isolation Forest
   - Local Outlier Factor (LOF)

3. **深度學習**
   - 圖神經網絡（GNN）
   - 時間序列模型（LSTM）

## 技術細節

### 為什麼 Recall 這麼低？

**訓練集統計**
```
正樣本（黑名單）：1,640 人
負樣本（正常用戶）：49,377 人
比例：1:30.1
```

**模型預測傾向**
- 模型傾向預測為負類（正常用戶）
- 即使設置 `scale_pos_weight=30`，仍然不足以平衡

**解決方案**
```python
# 增加正樣本權重
scale_pos_weight = 50  # 或更高

# 或使用 SMOTE 過採樣
from imblearn.over_sampling import SMOTE
```

### 為什麼只有 1 個預測正例？

**預測機率分布**
- 大多數用戶的預測機率 < 0.5
- 只有 1 個用戶的機率 > 0.5（user_id: 876703, 59.35%）

**改進方向**
1. 降低決策閾值到 0.3 或 0.2
2. 增加特徵維度
3. 改進特徵工程
